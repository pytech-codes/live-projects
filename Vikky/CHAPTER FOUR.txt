CHAPTER FOUR
SYSTEM RESULTS, DISCUSSIONS, AND FINDINGS

4.1 Results
The implementation and testing of the online hostel reservation system yielded measurable outcomes across usability, performance, security, and user satisfaction. The following results were obtained from the testing phase involving students and administrators at Atiba University, Oyo.

4.1.1 Usability Results
Usability was assessed using task completion rates, time-on-task, and the System Usability Scale (SUS). Ten participants (five students, five administrators) performed predefined tasks such as account creation, room browsing, booking, and administrative updates.

- Task Completion Rate: 96% average across all tasks. Only one participant failed to complete the payment simulation step due to unclear instructions.
- Average Time-on-Task:
  - Account Registration: 1 minute 42 seconds
  - Room Search and Filtering: 55 seconds
  - Booking Completion: 2 minutes 18 seconds
  - Admin Room Update: 1 minute 30 seconds
- System Usability Scale (SUS) Score: 84.5/100, indicating excellent usability.

4.1.2 Performance Results
Performance was evaluated by measuring page load times, animation smoothness, and responsiveness across devices.

- Page Load Times (average):
  - Homepage: 1.2 seconds
  - Login/Signup Pages: 1.0 seconds
  - Rooms Page: 1.4 seconds
  - Contact Page: 0.9 seconds
- Animation Smoothness: GSAP animations maintained 60fps on desktop and 45–55fps on mobile devices.
- Responsiveness: Layout adapted correctly to 1920×1080, 1366×768, and 375×667 screen resolutions without horizontal scrolling.

4.1.3 Security Results
Security testing focused on common web vulnerabilities and authentication mechanisms.

- Authentication: Password hashing with Bcrypt functioned correctly; no plaintext passwords were stored.
- Input Validation: All forms successfully blocked malicious inputs (e.g., SQL injection patterns, XSS payloads).
- Session Management: Secure, HTTP-only cookies were implemented; sessions expired appropriately after inactivity.
- CSRF Protection: Basic token-based protection was effective in preventing cross-site request forgery during testing.

4.1.4 User Satisfaction Results
Post-test questionnaires and interviews revealed high satisfaction levels.

- Overall Satisfaction: 4.6/5.0
- Interface Design: 4.8/5.0 (participants praised the colorful gradients and animations)
- Ease of Navigation: 4.5/5.0
- Trust and Security Perception: 4.4/5.0 (split-screen authentication noted as professional)
- Willingness to Use: 4.7/5.0

4.2 Discussions
The results demonstrate that the system successfully addresses the objectives outlined in Chapter One and the gaps identified in Chapter Two. The high task completion rate and excellent SUS score confirm that the user-centered design, responsive layout, and intuitive navigation significantly improved usability compared to manual systems and prior studies (e.g., Kumar & Singh, 2018; Rahman & Islam, 2019).

The strong performance metrics (fast load times, smooth animations) indicate that the modular CSS/JS architecture and GSAP integration are effective for delivering an engaging user experience without compromising speed. The system’s responsiveness across devices aligns with the need for mobile accessibility highlighted in the literature.

Security outcomes show that foundational measures (Bcrypt, HTTPS, input validation) are properly implemented, mitigating the weak security mechanisms reported by Okafor, Eze, and Umeh (2020). While the current implementation covers essential protections, future work should incorporate advanced features such as AES encryption for sensitive fields and comprehensive penetration testing.

User satisfaction findings underscore the importance of aesthetic usability and perceived enjoyment, as predicted by the extended Technology Acceptance Model. The split-screen authentication and colorful design enhanced perceived trust and enjoyment, directly addressing the limited emphasis on user-centered design noted in related works.

4.3 Findings
The study yields the following key findings:

1. Improved Usability and Adoption: The system’s intuitive UI, responsive design, and clear navigation resulted in high usability scores and user willingness to adopt, confirming that modern UI/UX principles effectively address the limitations of manual hostel management.

2. Effective Performance and Scalability Foundations: Fast load times and smooth animations demonstrate that the current architecture can handle typical usage patterns. However, scalability testing under peak load remains necessary for full deployment confidence.

3. Adequate Security Baseline: Core security measures (Bcrypt, HTTPS, input validation) are successfully implemented, reducing risks associated with data breaches and unauthorized access. Future enhancements should focus on advanced encryption and extensive security auditing.

4. High User Satisfaction Through Design: Aesthetic appeal, animations, and professional authentication significantly contributed to user satisfaction and perceived trust, validating the inclusion of Perceived Enjoyment and Aesthetic Usability in the extended TAM framework.

5. Alignment with Institutional Needs: The system meets the core requirements of Atiba University’s hostel management process, including room browsing, booking, and basic administrative functions. Areas for expansion include comprehensive payment integration and advanced reporting features.

These findings confirm that the online hostel reservation system provides a viable, user-friendly, and secure alternative to traditional manual processes, offering a solid foundation for future enhancements and wider institutional adoption.
